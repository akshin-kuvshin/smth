Есть две очереди. В первой стоят nn людей, во второй стоят mm людей. В первой очереди обслуживают одного человека за AA минут, во второй очереди обслуживают одного человека за BB минут. Отсчет времени начинается с минуты 11. Каждую минуту последовательно происходит следующее:

  1. Если первый человек в первой очереди обслужен, то он уходит из очереди.
  2. Если первый человек во второй очереди обслужен, то он уходит из очереди.
  3. Последний человек в первой очереди перемещается в конец второй очереди если его порядковый номер во второй очереди окажется меньше чем его номер в первой очереди.
  4. Последний человек во второй очереди перемещается в конец первой очереди если его порядковый номер в первой очереди окажется меньше чем его номер во второй очереди.

Сообщите первый момент времени TT когда все люди будут обслужены.

Входные данные

В первой и единственной строке входного файла даны четыре целых числа nn, mm, AA и BB (1≤n,m,A,B≤1051≤n,m,A,B≤105).

Выходные данные

Выведите одно целое число — первый момент времени TT когда все люди будут обслужены.

Минута 11. Первый человек из первой очереди обслужен и уходит из очереди. Теперь n=1n=1, m=3m=3. Последний человек во второй очереди переходит в конец первой очереди. Теперь n=2n=2, m=2m=2.

Минута 22. Первый человек из первой очереди обслужен и уходит из очереди. Теперь n=1n=1, m=2m=2. Первый человек из второй очереди обслужен и уходит из очереди. Теперь n=1n=1, m=1m=1. Больше ничего не происходит.

Дан ориентированный ацикличный граф из NN вершин и MM ребер. Вершины пронумерованы целыми числами от 11 до NN. Причем в графе нет ребра из вершины 11 в вершину NN.

Все вершины изначального белого цвета. Мансуру нужно покрасить ровно BB вершин в черный цвет. После покраски удаляются все ребра у которых оба конца одного цвета.

Помогите Мансуру покрасить BB вершин таким образом, чтобы нельзя было добраться из 11 в NN после удаления всех ребер у которых оба конца одного цвета.

Входные данные

Каждый тест содержит несколько наборов входных данных.

Первая строка содержит одно целое число TT(1≤T≤1001≤T≤100) — количество входных данных.

Первая строка каждого набора входных данных содержит три целых числа NN,MM и BB(3≤N≤1053≤N≤105, 0≤M≤min(2⋅105,N⋅(N−1)2)0≤M≤min(2⋅105,N⋅(N−1)2), 1≤B≤N1≤B≤N). Следующие MM строк каждого набора входных данных содержат по два целых числа uu и vv(1≤u,v≤N1≤u,v≤N, u≠vu≠v, (u,v)≠(1,N)(u,v)≠(1,N)) — задающие ориентированное ребро из вершины uu в вершину vv. Гарантируется, что в графе нет цикл и повторяющихся ребер.

Cумма NN по всем наборам входных данных не превосходит 106106.

Cумма MM по всем наборам входных данных не превосходит 106106.

Выходные данные

Если невозможно покрасить нужным образом, выведите «-1».

Иначе, выведите BB различных целых числа — номера вершин которые нужно покрасить в черный цвет. Если есть несколько возможных ответов, выведите любую из них.

Дан массив aa из nn целых положительных чисел. Для каждого целого числа kk от 11 до mm найдите значение (a1modk)⊕(a2modk)⊕…⊕(anmodk)(a1modk)⊕(a2modk)⊕…⊕(anmodk).

Здесь ⊕⊕ обозначает операцию побитового XOR или исключающего ИЛИ. Данная операция существует во всех современных языках программирования, например, в языках C++,Python и Java она обозначена как «^», в Pascal — как «xor».
